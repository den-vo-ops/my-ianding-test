## 1. Контекст

Статичный прототип (Tasks 1-10 предыдущего плана) собран и визуально оценён пользователем. По итогам оценки и нового запроса добавляются: визуальные «вау-эффекты», интерактивные элементы, и решается архитектурная проблема — главный JS-файл сейчас загружается как ES-модуль (`<script type="module">`), что браузеры блокируют при открытии файла напрямую (`file://`, без сервера) — а пользователю нужно, чтобы файл открывался корректно при получении через мессенджер.

## 2. Глобальные ограничения

- Никаких новых CDN-зависимостей (после случая с Tailwind CDN, не загрузившимся в окружении пользователя и сломавшим вёрстку — см. `prototype/css/utilities.css`).
- Все новые анимации отключаются под `prefers-reduced-motion: reduce`, как и существующие.
- Pointer-only эффекты (если появятся) — только при `(pointer: fine)`, как и существующий курсор-объект в Hero.
- Существующая структура 7 секций, контент и цены — без изменений.
- Поле формы «Telegram, Max или email» остаётся гибким (без переименования в строгий «email»).
- Тестовые файлы `prototype/tests/*.test.mjs` и их ES-модули (`js/magnetic.js`, `js/honeypot.js`, `js/kinetic-text.js`) — не меняются, остаются source of truth для логики.

## 3. file:// совместимость (главное архитектурное изменение)

**Проблема:** `<script type="module" src="js/main.js">` не выполняется при открытии `index.html` через `file://` (CORS-блокировка ES-модулей в браузерах) — главная причина, почему функциональность сайта не работает при открытии файла напрямую (например, полученного в мессенджере).

**Решение:** добавляется сборочный shell-скрипт `scripts/build-bundle.sh` (без npm/webpack/любых зависимостей) — склеивает содержимое `js/kinetic-text.js`, `js/magnetic.js`, `js/honeypot.js`, `js/main.js` в один файл `js/bundle.js`, вырезая строки `export `/`import ... from ...;` (классический скрипт, без модулей, делится глобальной областью видимости). `index.html` подключает `<script src="js/bundle.js"></script>` (без `type="module"`).

- Исходные 4 файла остаются неизменными и тестируемыми в Node как раньше.
- `bundle.js` — генерируемый файл с комментарием в начале: «Сгенерировано из js/*.js скриптом scripts/build-bundle.sh — не редактировать руками».
- После любого изменения исходных JS-файлов нужно перезапустить `scripts/build-bundle.sh`.

## 4. Визуальные «вау-эффекты»

### 4.1 Навигация + мобильное меню
- Sticky-шапка `<nav>` сверху страницы со ссылками на все 7 секций (`#hero`, `#cases`, `#services`, `#about`, `#testimonials`, `#faq`, `#contact`).
- Плавный скролл по клику — через CSS `scroll-behavior: smooth` на `<html>` (без JS), плюс `scroll-margin-top` на секциях, чтобы контент не прятался под шапкой.
- На экранах ≤768px — горизонтальный список ссылок заменяется гамбургер-иконкой; клик раскрывает вертикальное меню (slide/fade), клик по ссылке закрывает меню.
- Под `prefers-reduced-motion: reduce` — `scroll-behavior: auto` (мгновенный скролл вместо плавного).

### 4.2 Scroll-reveal анимация
- Каждая секция кроме Hero (`#cases`, `#services`, `#about`, `#testimonials`, `#faq`, `#contact`) получает fade-in-up при появлении в зоне видимости через `IntersectionObserver` (тот же паттерн, что уже используется для кинетического заголовка Hero).
- Под `prefers-reduced-motion: reduce` — секции видны сразу, без анимации.

### 4.3 Hover-эффекты
- `.glass-card:hover` — `translateY(-4px)` + усиление свечения в акцентном цвете (`box-shadow`), переход 0.2s ease.
- Кнопки (`.magnetic-btn` с акцентным фоном) — плавный переход `background-color` на hover (0.2s ease) в дополнение к существующему магнитному смещению.

### 4.4 Анимированный градиент заголовка
- Текст H1 в Hero — градиентная анимация в рамках текущей палитры: `--accent` (#7c5cff) ↔ `--accent-solid` (#6845e6) ↔ белый, через `background-clip: text` + keyframe-смещение `background-position`.
- Под `prefers-reduced-motion: reduce` — статичный сплошной цвет текста (`var(--text)`), без градиента и анимации.

### 4.5 Параллакс-эффект
- 2 декоративных размытых glow-пятна (`radial-gradient`, акцентный цвет, `filter: blur()`), `position: fixed`, `aria-hidden="true"`, `pointer-events: none`, за контентом (`z-index` ниже секций).
- Двигаются на части скорости скролла (коэффициент <1) через `scroll`-listener + `requestAnimationFrame` (без `--force-3d`/сторонних библиотек).
- Под `prefers-reduced-motion: reduce` — статичны, не двигаются (но остаются видимыми как декоративный фон).

### 4.6 Кнопка «наверх»
- Появляется (fade-in) после скролла примерно на высоту одного экрана.
- Клик — плавный скролл к началу страницы (тот же механизм `scroll-behavior: smooth`).
- Под `prefers-reduced-motion: reduce` — появляется без fade, скроллит мгновенно.

## 5. Интерактивные элементы

### 5.1 Анимация успеха формы
- При отправке формы (после текущей honeypot-проверки) форма скрывается, на её месте появляется SVG-галочка (анимация обводки через `stroke-dasharray`/`stroke-dashoffset`) + текст благодарности.
- Под `prefers-reduced-motion: reduce` — галочка появляется сразу, без анимации обводки.

### 5.2 Счётчики в Hero
- Новая строка из 3 плейсхолдер-счётчиков под основным CTA в Hero: «50+ проектов», «5 лет опыта», «14 дней поддержки» — с пометкой в коде `<!-- TODO: заменить на реальные цифры -->`.
- При появлении в зоне видимости (`IntersectionObserver`) число анимированно «считается» от 0 до целевого значения через `requestAnimationFrame` (~1-1.5 сек).
- Под `prefers-reduced-motion: reduce` — число показывается сразу в финальном значении, без анимации счёта.

## 6. Проверка на мобильных устройствах

Проверка через эмуляцию вьюпорта (375px, 768px) — чек-лист:
- Гамбургер-меню открывается/закрывается, ссылки кликабельны, меню закрывается после клика по ссылке.
- Кнопка «наверх» не перекрывает контент и не вылезает за пределы экрана.
- Счётчики и форма не ломают раскладку на узком экране.
- glow-пятна параллакса не вызывают горизontal-скролл (нет переполнения по ширине).
- Хедер не перекрывает контент секций при скролле по якорным ссылкам (`scroll-margin-top` работает).

По итогам — отчёт пользователю: что проверено, что работает, что не удалось проверить и почему (см. ограничения headless-тестирования в среде разработки).

## 7. Публикация на GitHub Pages

Вне рамок этой итерации — пользователь подтвердил, что хостинг не нужен, делимся самим файлом. Совместимость с `file://` (раздел 3) — единственный механизм надёжного открытия файла из мессенджера.

## 8. Вне рамок

- Превращение `bundle.js` в полноценный bundler-процесс (Webpack/Rollup/esbuild) — используется только простой shell-скрипт конкатенации.
- Реальные данные для счётчиков — остаются плейсхолдерами до появления настоящих кейсов.
- Реальная отправка формы на backend — анимация успеха остаётся чисто визуальной (как и было решено для Task 9).
- Изменение содержимого/структуры 7 существующих секций — только добавление новых визуальных слоёв и счётчиков, без правок текста.
- Публикация на внешний хостинг (GitHub Pages и т.п.) — пользователь подтвердил, что не нужна; результат — локальный файл `prototype/index.html`, открываемый напрямую.
