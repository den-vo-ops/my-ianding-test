### Сводный вывод

**Повторяющиеся паттерны-победители:**

1. **Hero = позиционирование + доказательство, без "обо мне" воды.** Везде в первом экране короткая фраза-заявление ценности, имя и фото — вторичны.
2. **Цифры и соцдоказательство встроены в hero или сразу после него**, а не в конце страницы: счётчики наград, опыт/проекты, логотипы клиентов, отзывы.
3. **Жёстко ограниченная палитра (2-4 цвета), почти всегда чёрно-белая база + один акцент.** Цвет — акцент-инструмент, не фон для иллюстраций.
4. **Единый скелет воронки:** Hero (хук) → доказательство (награды/клиенты/цифры) → портфолио кейсов → услуги/экспертиза (часто по доменам Frontend/Backend/AI/DevOps) → контакт/CTA.

**Шрифтовые пары, наиболее актуальные сейчас:**

- **Geist + Geist Mono** (Vercel) — встречен у 3 из 9 примеров независимо (Stackbyte, Cyd Stumpel, ForAI). Де-факто стандарт для tech/dev/AI-портфолио: нейтральный sans-serif + моно-вариант для кода/тегов/метрик. Бесплатный, легко встраивается в Next.js/Tailwind.
- **Neue Montreal + Roobert Mono** — у Mangham и Heymans. Более "дизайнерская" пара с характером.
- **Suisse Int'l + декоративный акцентный шрифт** — редкий "дизайн-студийный" паттерн.

**Рекомендация:** связка **Geist (текст/заголовки) + Geist Mono (код, теги стека, метрики)** на тёмном фоне с одним ярким акцентным цветом — ассоциируется с современным technical/AI-продуктом, бесплатна, подтверждена тремя независимыми кейсами.

## 2. Структура лучших лендингов

Анализ блочной структуры на основе onepagelove.com, tilda.cc/tildapublishing и конкретных кейсов (stackbyte.dev, joegarnerdesign.com, garriock.com, carlos-molina, indiez).

### 2.1 Что нашли

**stackbyte.dev:** Hero → About/My Story → Experience Timeline → Process (Contact → Plan → Develop → Deploy) → Services (по стекам) → Project Showcase (open source) → финальный CTA → Footer.
*Работает:* конкретный 4-шаговый процесс снимает неопределённость; pet-проект как доказательство экспертизы.

**joegarnerdesign.com:** Header (теги услуг) → Hero → About ("luxury brands") → Featured Works → Process (Discovery → Direction → Design → Development) → FAQ (8 вопросов) → Footer.
*Работает:* нишевание отсекает нецелевых лидов и повышает цену; FAQ перед футером закрывает последние возражения.

**garriock.com:** Hero ("Design leadership... on demand") → What We Do → How We Work (модель "под проект") → Service Offerings (6) → The Partners (логотипы Apple/Microsoft/Nike/Adobe) → Q&A (4) → CTA с прямой ссылкой на календарь → Footer.
*Работает:* логотипы клиентов сразу после объяснения модели работы; CTA — прямая ссылка на календарь, минимум трения.

**carlos-molina (onepagelove):** полноэкранный hero с фото, плавные переходы, кастомный прелоадер, параллакс, качественная форма.
*Работает:* для визуальных профессий сама эстетика — демонстрация навыка.

**indiez (onepagelove):** Hero → инфографика "как это работает" (с эмодзи) → яркие блоки с крупными изображениями.
*Работает:* визуальный сторителлинг снижает когнитивную нагрузку перед конверсией.

**Анатомия Tilda (28 типовых блоков, обобщённая модель):** обложка → о проекте → преимущества → выгода/сценарии → ЦА → тарифы → CTA → форма → отзывы → партнёры → галерея/портфолио → этапы работы → FAQ → контакты → команда → факты в цифрах.
*Работает:* строгая последовательность "проблема → решение → доказательство → действие"; CTA дублируется в нескольких точках скролла.

### 2.2 Идеальная структура (7 блоков)

#### 1. Hero (первый экран)
**Цель:** за 3 секунды ответить "кто это и зачем мне с ним работать".
**Внутри:**
- Заголовок-позиционирование с конкретикой результата: *"Делаю сайты, приложения и AI-агентов, которые приносят клиентам деньги"*
- Подзаголовок с уточнением стека (Next.js, Python, LLM-агенты)
- Главный CTA: "Обсудить проект" → ссылка на календарь, не mailto
- Превью лучшего кейса (мокап/видео) или короткое личное видео 15-20 сек
- Строка соцдоказательства под CTA: "50+ проектов · 5 лет на фрилансе · [логотипы]"

#### 2. Доказательство экспертизы — портфолио/кейсы (3-5 проектов)
**Цель:** перевести "я хорош" в "вот что я сделал и какой результат".
**Внутри:**
- Формула "Проблема → Решение → Результат в цифрах"
- Минимум один кейс — именно AI-агент (differentiation)
- Скриншот/мокап + ссылка "Смотреть проект" или живая демка
- Разнообразие типов (сайт / приложение / AI-агент)

#### 3. Услуги и как я работаю (process)
**Цель:** снять неопределённость "как будет проходить работа" — главный барьер перед обращением.
**Внутри:**
- 3-4 направления: "Сайты и лендинги", "Приложения", "AI-агенты и автоматизация", "Поддержка"
- Процесс из 4 шагов со сроками: *"1. Звонок (30 мин) → 2. Бриф и оценка → 3. Разработка с демо каждую неделю → 4. Запуск и поддержка"*
- Явные сроки и формат оплаты

#### 4. О себе (About) — короче, чем кажется нужным
**Цель:** очеловечить, закрыть "почему я, а не агентство".
**Внутри:**
- 3-4 предложения: путь, специализация, отличие
- Реальное фото (не сток)
- 2-3 факта в цифрах
**Почему здесь:** после доказательств и процесса, не перед ними — ошибка многих ставить About первым.

#### 5. Отзывы / социальное доказательство
**Цель:** нейтрализовать последнее рациональное сомнение.
**Внутри:**
- 3-5 текстовых отзывов с фото/именем/должностью, с конкретным результатом в цитате
- Если нет отзывов — логотипы компаний, с которыми работал
- Опционально: ссылка на профиль с рейтингом (Upwork, GitHub stars)

#### 6. FAQ — закрытие возражений
**Цель:** убрать стоп-факторы, специфичные для AI-агентов и фриланса.
**Внутри (5-7 вопросов):**
- Сколько стоит разработка AI-агента/сайта?
- Сколько времени занимает проект?
- Что если нет ТЗ?
- Кастомный код или no-code/AI-инструменты?
- Как оплата и есть ли гарантии?
- Есть ли поддержка после запуска?

#### 7. Финальный CTA + контакты
**Цель:** максимально простое действие в момент максимального доверия.
**Внутри:**
- Заголовок: *"Готовы обсудить проект?"*
- Главная кнопка — прямая ссылка на календарь (Calendly/Cal.com)
- Альтернативный контакт: Telegram/WhatsApp (для рунета часто конвертит лучше формы)
- Короткая форма как запасной вариант
- Footer: соцсети/портфолио-профили (GitHub, Behance, LinkedIn), копирайт

**Общий принцип:** Внимание (hero) → Доказательство (кейсы) → Понимание процесса → Личная связь (about) → Соцдоказательство (отзывы) → Снятие возражений (FAQ) → Действие (CTA). Каждый блок отвечает на вопрос, рождённый предыдущим блоком.

## 3. Тренды и фишки 2026 года

На основе awwwards.com, css-tricks.com и смежных источников (Smashing Magazine, Figma, dev.to, Envato).

### Анимации и интерактив
- Скролл-анимации (scroll-driven CSS, GSAP ScrollTrigger) — главный тренд, "кинематографичный" browsing
- Кинетическая типографика — текст реагирует на скролл/курсор
- Курсор-эффекты — кастомные курсоры, magnetic-кнопки
- Лёгкий 3D/WebGL — точечные тачи вместо тяжёлых сцен
- Микро-взаимодействия на hover/click/submit
- Pure CSS вместо JS (`@starting-style`, scroll-driven animations, view-transitions)
- Смешанные направления скролла (вертикаль + горизонталь для отдельных секций)

### Цветовые схемы и стиль
- Dark mode + Glassmorphism ("Liquid Glass") — тёмный фон, blur-карточки, неоновые орбы
- Neo-brutalism — грубые контрастные блоки, явные сетки
- Яркие палитры/неон — возврат Y2K-эстетики
- "Спокойная база + один яркий акцент"
- Текстуры, шум, glitch в сочетании с крупной типографикой

### Типографика
- Variable fonts (анимируемые вес/ширина через `font-variation-settings`)
- Kinetic typography — текст как UI-элемент
- Огромные заголовки (oversized type)
- Сломанные/асимметричные грид-layouts

### Прочие фишки
- AI как часть интерфейса — проактивные чат-боты/агенты на сайте
- Персонализация на основе поведения
- Sonic UX — звуковые сигналы как часть бренда
- "Человеческий" арт-дирекшн как сигнал доверия (контраргумент AI-generic дизайну)
- Performance-first дизайн — минимизация веса страницы

### ТОП-5 трендов, применимых для лендинга разработчика/AI-специалиста

#### 1. Scroll-driven кинетическая типографика
**Суть:** заголовки оживают при скролле — буквы/слова появляются, растягиваются, меняют вес/цвет по позиции скролла, без тяжёлого 3D.
**Как применить:** в hero — заголовок "Разрабатываю сайты, приложения и AI-агентов", где слова подменяются/"собираются" по буквам при загрузке/скролле. Технически: `animation-timeline: scroll()` (CSS) + GSAP `ScrollTrigger`/`SplitText` как фолбэк.

#### 2. Тёмная тема + Glassmorphism с неоновым акцентом
**Суть:** тёмный фон, полупрозрачные blur-карточки, 1-2 светящихся акцентных цвета для CTA и иконок.
**Как применить:** фон `#0a0a0f`, карточки услуг/проектов — `background: rgba(255,255,255,0.05); backdrop-filter: blur(16px); border: 1px solid rgba(255,255,255,0.1)`. Акцент — светящаяся обводка у CTA-кнопки. Реализуется чистым CSS.

#### 3. Легковесные 3D/курсор-эффекты вместо полноценного WebGL
**Суть:** не киношные 3D-сцены, а точечные тачи — объект, слегка вращающийся за курсором.
**Как применить:** в hero — один 3D-объект (нейросеть/код-блок) через Three.js/`@react-three/fiber`, parallax 5-10° по `mousemove`. Лёгкая альтернатива: `transform: perspective() rotateX() rotateY()` через несколько строк vanilla JS — 80% эффекта за 5% усилий.

#### 4. Микро-взаимодействия с мгновенным фидбеком
**Суть:** каждая кнопка/карточка реагирует на hover/клик заметной анимацией.
**Как применить:** "magnetic button" на CTA (смещение к курсору в радиусе 20-30px через `transform: translate()` + `transition: 0.2s ease-out`); на карточках проектов — подъём (`translateY(-4px)`) + усиление тени/glow на hover. Без сторонних библиотек.

#### 5. AI как видимая часть лендинга
**Суть:** вместо текста "я делаю AI-агентов" — встроенное интерактивное мини-демо/чат-виджет, который посетитель может попробовать на сайте.
**Как применить:** в секции "AI-агенты" — рабочий чат-виджет (Claude/OpenAI API через serverless-эндпоинт), отвечающий о портфолио/услугах в стиле личного ассистента. Технически: React/vanilla-компонент чата + serverless function (Vercel/Cloudflare Workers) к LLM API. Посильно за 1-2 дня одному разработчику и прямо доказывает компетенцию в AI лучше любого текста.

## 4. Экспертная оценка

Аудит уже выбранной структуры (7 блоков) и визуального стиля (тёмная тема, glassmorphism, Geist/Geist Mono, scroll-анимации, 3D-курсор, magnetic-кнопки) с трёх точек зрения. Каждый эксперт проверял факты через веб-поиск, а не давал оценку "на глаз".

### 4.1 UX-дизайнер (10 лет опыта)

**Типичные ошибки новичков в портфолио-лендингах** (и риски для нашей структуры):
1. Слишком много или слишком мало кейсов — норма 3-5 (NN/G). У нас 3-5 — попадание в норму.
2. Кейсы-«скриншоты» без истории — антидот: формула "Проблема → Решение → Результат в цифрах", но риск в придуманных/неправдоподобных цифрах результата — это подрывает доверие сильнее, чем их отсутствие.
3. Неочевидная ценность в первом экране — решение о доверии формируется за 50 мс (исследование Линдгаард). Риск: если scroll-driven анимация заголовка "доанимируется" слишком медленно, эта решающая секунда уходит на ожидание текста, а не на восприятие ценности.
4. Несогласованность дизайна между кейсами и общим стилем — следить, чтобы скриншоты чужих (часто светлых) интерфейсов не "спорили" с тёмным фоном лендинга.
5. Устаревший контент — нет механизма поддержания свежести (даты кейсов, "обновлено в...").
6. Жаргон — стек технологий в hero ("Next.js, LangChain, RAG") может работать как шум для нетехнического владельца бизнеса.
7. Скрытая/неочевидная CTA — насыщенные 3D/scroll-эффекты могут визуально "потерять" кнопку записи на звонок.

**Чего не хватает в структуре (технический и a11y-слой, не покрытый контентными блоками):**
- **Производительность — главный конфликт проекта.** Конверсия падает на ~4.4% за каждую доп. секунду загрузки (0-5 сек), bounce растёт на 32% при росте загрузки с 1 до 3 сек и на 106% при росте до 6 сек (Outerbox Design). На мобильных задержка в 1 сек может снижать конверсию до 20%. Только 48% мобильных сайтов проходят все три Core Web Vitals одновременно (2026). 3D-курсор и scroll-эффекты почти гарантированно бьют по LCP/INP/CLS без оптимизации.
- **`prefers-reduced-motion`** — обязателен по WCAG 2.3.3, не опция (вестибулярные нарушения от движения).
- **Контраст текста в glassmorphism-карточках** — полупрозрачность+блюр — частый источник провала по WCAG (4.5:1) из-за переменного фона под текстом; нужно тестировать на реальных скриншотах, не "на глаз".
- **Защита формы от спама** (honeypot + invisible CAPTCHA/Turnstile) — не упомянута в структуре, без неё форма зарастёт спамом за недели.
- **Альтернативный путь конверсии без JS** — CTA на календарь/Telegram должны быть обычными `<a>`-ссылками, не зависящими от инициализации анимационного движка.
- Семантика заголовков (h1-h2-h3), alt-тексты, focus-состояния для клавиатуры, базовое SEO (meta, OG-теги).

**Антипаттерны, из-за которых уходят, и риски конкретных "модных" фишек проекта:**

| Фишка | Риск стать антипаттерном | Как избежать |
|---|---|---|
| Scroll-driven типографика | Медленная анимация = текст физически нельзя прочитать на скролле — похоже на scrolljacking, который NN/G напрямую связывает с дезориентацией пользователей | Длительность 300-500мс, никогда не менять физическую скорость/направление скролла, `prefers-reduced-motion` фоллбэк |
| 3D/курсор-эффекты | На мобильных (без курсора) бесполезны, но если тянут тяжёлый WebGL — чистый минус к LCP без пользы | Рендер только при `(pointer: fine)`, lazy-load после первого экрана |
| Тёмная тема | Риск проблем с контрастом и нагрузкой на глаза при длинных абзацах (FAQ, кейсы) | WCAG AA 4.5:1 на каждом блоке, длинный текст — не белым по чёрному, а приглушённым серым |
| Glassmorphism-карточки | Главный источник провала контраста; `backdrop-filter: blur()` дорог по производительности на слабых GPU | Тест контраста на реальном скриншоте; fallback без blur для слабых устройств |
| Magnetic-кнопки | Большая зона "притяжения"/медленная реакция = кнопка ощущается неотзывчивой | Реакция ≤150-200мс, не применять на мобильных |

**Вердикт UX-эксперта:**
1. Зафиксировать перформанс-бюджет (LCP < 2.5с) до вёрстки, не после.
2. Hero должен быть читаемым и кликабельным до полной инициализации JS-анимаций (progressive enhancement).
3. `prefers-reduced-motion` + условный рендер 3D по `(pointer: fine)` — обязательное требование, не nice-to-have.
4. Прогнать контраст текста в glassmorphism-карточках через чекер на реальном фоне с блюром перед запуском.
5. Защита формы от спама (honeypot + Turnstile) в блоке 7.
6. Стек технологий — убрать из hero как шум, перенести в блок "Услуги"/"О себе", где есть контекст.
7. Микро-фидбэк состояний (hover/focus/active, успех/ошибка формы) для всех интерактивных элементов.
8. Критично: CTA на календарь/Telegram должны работать как обычные ссылки независимо от загрузки анимационного движка.

### 4.2 Маркетолог (специалист по конверсии)

**Факты:** landing page для услуг/агентств конвертит в среднем 8.8%, для B2B — 13.28% (LanderLab); репозиционирование hero с абстрактного на конкретное снизило bounce с 67% до 38% и поднял переходы к заявке на 45% (Atticus Li); видимые трасты повышают конверсию на 42%, соцдоказательство под CTA — на +68% (Crazy Egg, Amra & Elma); смена CTA с «Book a Demo» на «Get Started» дала +111% конверсии.

**Какой оффер зацепит аудиторию (владельцы малого/среднего бизнеса):** аудитория покупает не "сайт", а решение операционной боли + страх подрядчика, который пропадёт/сорвёт сроки. Формула: [для кого] + [измеримый результат] + [срок/цифра], не описание категории услуги. Примеры:
- «Делаю сайты и AI-агентов, которые приносят заявки, а не просто красиво лежат в портфолио»
- «AI-агент, который отвечает клиентам за 30 секунд вместо 24 часов — без найма менеджера»
- «Сайт под ключ за 14 дней: от брифа до первой заявки клиента»
- «10 сайтов и AI-агентов, 0 просроченных дедлайнов, средний рост заявок — 35%»
- «Не дизайнер. Не агентство. Один человек, который доводит сайт/агента до результата и потом не пропадает»

**Триггеры доверия, специфичные для ниши "соло-фрилансер + AI":**
- Цифры в отзывах, а не эмоции (имя+должность+компания+метрика, не "всё понравилось")
- ROI в кейсах, особенно в AI-кейсе — конкретная цифра экономии времени/денег, иначе AI-кейс выглядит как технодемо, а не бизнес-инструмент
- Прозрачность цены — хотя бы вилки, отсутствие ориентира по цене — частая причина отказа на этапе FAQ
- Гарантия снижения риска первого шага (money-back на первый этап, бесплатная диагностика до оплаты — аналог практики 99designs)
- Явная политика поддержки после сдачи — закрывает страх "пропадёт после оплаты", специфичный именно для соло-исполнителя (у агентства есть иллюзия "команды")
- Реальное лицо, не стоковое фото
- Конкретный AI-стек (на базе GPT-4/Claude, интеграция с CRM) — снимает ощущение "это просто чат-бот за 500 рублей"

**Путь посетителя (воронка по блокам 1-7) с drop-off на каждом шаге** — подробно описан агентом; ключевой вывод: самый слабый элемент структуры — блок 5 (отзывы) + блок 6 (FAQ), потому что именно там клиент решает "доверять ли одному человеку, а не агентству" — а соло-ниша конвертит ниже среднего именно из-за этого "парадокса доверия".

**Вердикт маркетолога:**
1. Проверить, что заголовок hero — оффер (для кого+результат+срок), а не категория услуги.
2. AI-кейс обязательно с цифрой ROI/экономии времени, иначе выглядит как игрушка.
3. FAQ должен прямо закрывать три вопроса: цена (хотя бы вилка), "что если не понравится результат" (гарантия), "что после сдачи" (поддержка).
4. Блок 3 — конкретные сроки по этапам, а не "анализ → разработка → запуск".
5. Добавить гарантию снижения риска первого шага.
6. Встроить микроконверсии разной "температуры" в каждый блок (скачать кейс, смета-калькулятор, написать в Telegram), не только финальную форму.
7. CTA — конкретная формулировка действия ("Забронировать 15-минутный разбор бесплатно"), не "связаться с нами".
8. Усилить отзывы (цифры+фото+должность) и FAQ — это решающая точка для доверия к одному человеку.

### 4.3 Потенциальный клиент (владелец бизнеса, нетехнический)

**Факты:** оценка дизайна сайта формируется за 50мс и сильно коррелирует с оценкой через 500мс (Google Research); вероятность отказа растёт на 32% при росте загрузки с 1 до 3 сек, на 90% — до 5 сек (Pingdom/Huckabuy); 76% B2B-покупателей требуют соцдоказательство перед решением, 92% читают отзывы перед покупкой услуги.

**Первые 5 секунд:** взгляд скользит мимо 3D-эффекта к заголовку — нужно понять "это тот, кто мне нужен?". Цифры ("50+ проектов") считываются и успокаивают. Подзаголовок с "Next.js, Python, LLM-агенты" — не читается, это шум для нетехнического человека (деталь двигателя для того, кто просто хочет, чтобы машина ехала). Если в первые секунды нет ни одного кейса, похожего на свою отрасль — лёгкий скепсис: "ещё один фрилансер с красивой анимацией".

**Останусь / уйду:**
- Останусь, если вижу превью реального кейса с понятным бизнес-результатом, кейс близкий к своей сфере, конкретные цифры про деньги/время, понятный следующий шаг.
- Уйду, если сайт долго грузится (прямое противоречие: "продаёшь скорость, а сам тормозишь"), анимации мешают читать, технический жаргон не переведён на язык результата.
- Тёмная тема/3D/glassmorphism — не "несерьёзно" сами по себе (это демонстрация технологичности), НО красота должна продавать, а не тормозить — если анимации замедляют поиск кейсов/цены, это работает против.

**Что мешает оставить заявку (нерешённые сомнения):**
1. "Понимает ли он мой бизнес или только технологии?" — нужен кейс из близкой отрасли с бизнес-метрикой, не "сделал лендинг на React".
2. "Сколько это стоит хотя бы примерно?" — полное молчание о цене = уход "подумать" без возврата.
3. "А вдруг пропадёт после оплаты?" — закрывается отзывами с реальными именами/фото/должностями, желательно со ссылками на профили.
4. "Что если что-то пойдёт не так после запуска?" — пункт про поддержку должен быть явным.
5. "Сколько займёт по времени?" — нереалистично короткие сроки вызывают обратный эффект (недоверие к качеству).

**Вердикт клиента:**
1. Убрать/перевести технический жаргон в hero на язык бизнес-результата, стек — мелким текстом ниже.
2. Первый кейс — максимально "обычный" бизнес, не только модный AI-агент.
3. Указать хотя бы ориентир цены в FAQ.
4. Сайт быстрее, чем красивее — загрузка 1-2 сек, иначе обратный эффект сильнее "вау"-эффекта.
5. Отзывы — с фото, именем, должностью, ссылкой на профиль/компанию.
6. В блоке "Услуги" явно прописать условия поддержки и что происходит при проблемах после оплаты.
7. Кейсы — буквально по формуле "проблема-решение-результат", цифра результата крупным шрифтом.
8. Финальный CTA дублировать несколько раз по странице, включая кнопку Telegram/WhatsApp — решение об заявке часто спонтанное, в момент убеждения, не на последнем скролле.

### Общий список рекомендаций (синтез трёх экспертиз)

**Критичные риски (могут провалить конверсию):**
- Производительность vs заявленные 3D/scroll-эффекты — главный системный конфликт проекта, отмечен всеми тремя экспертами с разных сторон (UX: Core Web Vitals; маркетолог: bounce rate; клиент: "продаёшь скорость, а сам тормозишь"). Решение: perf-бюджет LCP < 2.5с, progressive enhancement, lazy-load анимаций, `prefers-reduced-motion`.
- Технический жаргон в hero (стек технологий) — мешает нетехническому владельцу бизнеса понять оффер. Перенести в блок "Услуги"/"О себе", в hero — язык бизнес-результата.
- Отсутствие цены/вилки в FAQ — частая причина ухода "подумать" без возврата.
- Отзывы без цифр/фото/должности не работают как доказательство — нужно явное ТЗ на сбор отзывов с метриками.
- AI-кейс без конкретного ROI выглядит как технодемо, а не бизнес-инструмент.

**Обязательно добавить в структуру:**
- Гарантия снижения риска первого шага (money-back на этап / бесплатная диагностика).
- Явный пункт о поддержке после сдачи проекта (что входит, как быстро отвечает).
- Защита формы от спама (honeypot + Turnstile).
- Микроконверсии разной "температуры" по всем блокам (скачать кейс, смета-калькулятор, переход в Telegram), не только финальная форма.
- Альтернативный путь конверсии без JS — CTA-ссылки должны работать независимо от анимационного движка.
- Контраст текста в glassmorphism-карточках — тест по WCAG AA на реальном фоне с блюром.

**Усилить:**
- Заголовок hero — превратить из категории услуги в конкретный оффер (для кого + результат + срок).
- Минимум один кейс из "обычной" непримечательной отрасли бизнеса, не только AI-агент.
- Сроки этапов процесса — конкретные дни/недели, не абстрактные "анализ → разработка".
- CTA-формулировки — конкретное действие с выгодой ("Забронировать 15-минутный разбор бесплатно"), не "связаться с нами".
